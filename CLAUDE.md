## Início de Sessão

1. Leia `docs/progresso.md` e `docs/mapa-do-sistema.md`
2. Identifique a fase atual do projeto
3. Apresente resumo e pergunte o que fazer

## Fim de Sessão

1. Atualize `docs/progresso.md`
2. Crie/atualize `docs/diario/YYYY-MM-DD.md`
3. Mova tasks concluídas para `docs/tasks/concluidas.md`
4. Commite tudo no git

## Regras Críticas

- **IMPORTANTE:** Pergunte antes de avançar de fase
- **IMPORTANTE:** Ao receber ideia ambígua, pergunte: "Fazer agora ou registrar como ideia?"
- **IMPORTANTE:** Ao receber feature nova durante implementação, pergunte: "Isso é MVP ou futuro?"
- Atualize `docs/mapa-do-sistema.md` após cada feature implementada
- Antes de modificar código, leia `docs/mapa-do-sistema.md` para entender impacto
- Atualize este arquivo quando identificar melhorias nas regras
